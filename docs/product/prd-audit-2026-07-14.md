## Part A вЂ” Spec vs. shipped build (substantive gaps)

Ranked by how much they undermine the PRD's own pillars.

### S1 (P0) вЂ” The music the player hears is not the beat the game judges

В§7.3 pillar 3 ("the arena beats to the trackвЂ¦ reading the music is reading the fight"),
В§8.3, В§10.3 ("Battle music: authored stem playback aligned to bar/beat markers"), and
release-gate framing all assume the audible music and the judgment clock are one thing.
They are not:

- `SongPlayer.ts` streams the six MP3s via `HTMLAudioElement` with **no tempo map, no
  bar markers, and no connection to `TransportClock`**. Songs start whenever the fade
  starts and loop at file length.
- `WorldFight.ts:136-148` derives the judged beat from the *beatmap's* BPM on
  `Tone.Transport`, and the only audible signal on that actual beat is the sonifier
  tick deliberately ducked to Г—0.22 (v7.7) so it doesn't clash with the music.
- Worse, game speed multiplies the judgment BPM (`WorldFight.ts:138`) but not the
  MP3's `playbackRate` вЂ” at 70%/85% speed the divergence between heard music and
  judged beat *grows*, in the accessibility mode meant to make timing easier.

So the shipped build plays music *over* the fight while judging against an inaudible
metronome. That is precisely the "timing is hidden" failure mode В§3's research findings
warn kills this genre. Either the songs need real tempo/bar maps aligned to the
transport (the В§11.2 spec, unmet for all six tracks вЂ” `assets/audio/stems/` is an empty
directory), or В§7.3/В§8.3/В§10.3/В§11.2 need rewriting to describe what "on-beat" means
when the beat is not the song's.

### S2 (P0) вЂ” В§8.7 final boss (3 phases, live meter changes) is absent from the shipped combat path; release gate #3 is unsatisfiable

`WorldFight.ts` has a boss bar and boss music but **no phase logic and no meter
changes** вЂ” `MeterSequence` and `BossPhaseConfig` are consumed only by the retired
turn-based `BattleScene`. The e2e spec that "proves" the 3-phase mechanic
(`tests/e2e/boss-phases.spec.ts`) still drives `BattleScene`, a scene no player can
reach from the shipped trigger path (retired v7.13). Release gate 3 (В§16.2: "final boss
reliably executes authored meter changes") cannot currently pass against the game as
played. В§8.7 needs either an action-path implementation or explicit re-spec.

### S3 (P0) вЂ” В§8.3's four judgment tiers don't exist in the shipped combat

В§8.3 states "the four judgment tiers now grade the timing of a real-time action" with a
Perfect/Great/Good/Off table (В±45/В±90/В±140 ms). The shipped fight has a **single binary
window**: `WorldFight.ts:208-211` returns on-beat iff offset < 90 ms (Г—1.5 assist).
No Perfect tier, no Good tier, no graded bonuses. `JudgmentSystem`'s four tiers live
only in the retired path. Either implement the table or rewrite В§8.3 to the binary model
actually shipped (and re-check В§5's "Miss rate on Perfect/Great judgments" KPI, which
is currently unmeasurable вЂ” see S6).

### S4 (P1) вЂ” Ultimates/Groove-spend regressed вЂ” the exact bug v5.1 fixed, reintroduced by the pivot

v5.1's headline fix was "В§8.5's Groove had no spend path at all." The action sim has
regressed to precisely that state: `ActionCombat.ts:25` says "**ultimate is future**",
Groove only accumulates (`:243`, `:265` вЂ” parry +14, hits +8), and nothing can spend
it. В§8.2 specs Ultimate as one of the four attacks; the v7.11 HUD renders a Groove bar
that can never be used. The PRD nowhere admits this regression вЂ” В§20.4 lists the
"full *Melee* tech tree" as still-being-built but not the missing ultimate.

### S5 (P1) вЂ” В§8.4 per-hero movesets: not built, and the named cast no longer matches the shipped content

The shipped sim has **one** shared `PLAYER_ATTACKS` moveset. The four В§8.4 identities
(the Deereater's lunge, the Saltminer's wide parry+counter, the Esoterophobe's hex
zone, Sunshine Sally's Sightread + heal pulse) exist nowhere in the action path. And the
hero content that ships names the party **Amir / Bassist / Vocalist / Drummer**
(`src/data/content/heroes/*.json`) вЂ” В§8.4's song-title names were never reconciled with
the v7.9 band cast. Related: **Sightread вЂ” the realization of the source concept's "see
the music," a В§5 KPI, and a В§14 event вЂ” is entirely absent from the shipped combat**
(present only in retired `BattleScene`/`Forecast.ts`).

### S6 (P1) вЂ” Mandatory В§9.3 accessibility items are missing from the shipped combat path

В§9.3 is "mandatory, day-one," and В§20.1 claims every setting is "functionally wired."

### S7 (P2) вЂ” В§5 KPIs and В§14 analytics are largely unmeasurable in the shipped game

`judgment_perfect`, `judgment_miss`, `ability_used`, `sightread_used`, and
`boss_phase_reached` fire only in the retired path. Four of В§5's eight KPIs (difficulty
calibration, Sightread usage, boss phase-3 reach, and effectively the assist-adoption
